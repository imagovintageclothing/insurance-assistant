# WORKFLOW: SVILUPPO APP GESTIONE PERIZIE ASSICURATIVE

**Obiettivo:** Creare una Web App privata, accessibile da mobile, per la compilazione automatica di perizie tramite LLM, sincronizzata con un database cloud (Supabase) e con archiviazione sicura dei documenti su Google Drive.

---

## FASE 1: SETUP DELL'AMBIENTE E DEL DATABASE (Giorno 1)
*L'obiettivo di questa fase è preparare le fondamenta del progetto dove verranno salvate le pratiche.*

1. **Creazione Database su Supabase:**
   * Registrati su [Supabase](https://supabase.com/) (Piano Free).
   * Crea un nuovo progetto e apri l'editor SQL.
   * Esegui il comando per strutturare la tabella `pratiche`:
     * `id` (Chiave primaria, autoincrementale)
     * `data_creazione` (Timestamp automatico)
     * `nome_cliente` (Testo)
     * `targa` (Testo)
     * `modello_auto` (Testo)
     * `assicurazione` (Testo)
     * `dinamica_breve` (Testo/Area di testo)
     * `relazione_ia` (Testo esteso generato dal modello)
     * `google_drive_id` (Testo, codice univoco del file)
     * `stato` (Testo, valore predefinito: "Aperta")

2. **Setup dell'Ambiente Locale:**
   * Installa Python sul tuo computer attuale.
   * Crea una cartella di progetto dedicata sul tuo PC (es. `app-perizie`).
   * Apri il terminale all'interno della cartella e installa le librerie iniziali:
     ```bash
     pip install streamlit supabase openai
     ```

---

## FASE 2: CONFIGURAZIONE DELL'IA E DEI PROMPT (Giorno 2)
*Configurazione del motore di intelligenza artificiale in cloud tramite API gratuite al consumo.*

1. **Ottenere le Chiavi d'Accesso:**
   * Registrati su [Groq Cloud](https://console.groq.com/) o [OpenRouter](https://openrouter.ai/).
   * Genera una **API Key** privata e salvala provvisoriamente in un file di testo locale.

2. **Ingegneria del Prompt (Prompt Engineering):**
   * Definisci lo **"Scheletro"** del documento peritale (un file di testo standard con tag come `[TARGA]`, `[RELAZIONE_TECNICA]`, ecc.).
   * Scrivi il *System Prompt* per l'LLM (es. `llama-3.1-8b-instant` o `qwen-2.5-coder-7b`):
     > *"Sei un perito assicurativo senior. Prendi i dati forniti e la dinamica breve e compila lo scheletro del testo espandendolo in un italiano formale, tecnico e peritale. Non aggiungere saluti o testi introduttivi."*
   * Testa il comportamento del modello direttamente nel Playground della piattaforma scelta.

---

## FASE 3: SVILUPPO DELL'APPLICAZIONE IN LOCALE (Giorno 3-5)
*Scrittura del codice Python con Streamlit e verifica delle funzioni sul computer di casa.*

1. **Pagina 1: Inserimento e Generazione (Form):**
   * Crea l'interfaccia utente con i campi di inserimento (`st.text_input` per i dati, `st.text_area` per la descrizione).
   * Configura il bottone "Genera": al click, l'app invia i dati a Groq/OpenRouter, riceve il testo formale dall'LLM e lo mostra a schermo in anteprima.

2. **Integrazione con Supabase (Salvataggio):**
   * Scrivi la funzione di inserimento dati. Quando l'anteprima è corretta, l'app invia una query a Supabase per inserire una nuova riga con tutti i dati del form e il testo dell'IA.

3. **Pagina 2: Dashboard e Modifica:**
   * Crea una seconda scheda/pagina nell'app che interroga Supabase e mostra la tabella di tutte le pratiche attive.
   * Fai in modo che cliccando su una pratica contrassegnata come "Aperta", i dati vengano ricaricati nei campi del form per permettere modifiche manuali o ricalcoli.

---

## FASE 4: INTEGRAZIONE CON GOOGLE DRIVE (Giorno 6)
*Scollegare la dipendenza esclusiva dal DB salvando una copia fisica dei file in una cartella sicura.*

1. **Configurazione Google Cloud:**
   * Accedi alla Google Cloud Console e crea un progetto gratuito.
   * Abilita le **Google Drive API**.
   * Crea un **Service Account** (credenziali robot per l'app) e scarica il file delle chiavi in formato `.json`.
   * Vai sul tuo Google Drive personale, crea una cartella (es. `Archivio_Perizie`) e condividila con l'indirizzo email del Service Account (permessi di scrittura).

2. **Codice di Sincronizzazione File:**
   * Installa le librerie Google sul PC:
     ```bash
     pip install google-api-python-client google-auth-httplib2 google-auth-oauthlib
     ```
   * Modifica il flusso dell'app: quando la perizia viene confermata, il codice genera un file di testo/PDF, lo carica su Google Drive tramite API, recupera il `File ID` univoco e lo salva nella colonna corrispondente su Supabase.
   * Aggiungi un bottone "Visualizza Documento" nella Dashboard per scaricare e aprire il file direttamente da Drive sul telefono.

---

## FASE 5: SICUREZZA E DEPLOY ONLINE (Giorno 7)
*Blindare l'applicazione per renderla accessibile solo ed unicamente a te prima di pubblicarla.*

1. **Autenticazione Integrata:**
   * Installa la libreria di sicurezza: `pip install streamlit-authenticator`.
   * Configura una password complessa (protetta da hash). Inserisci il blocco in cima allo script: se le credenziali non sono corrette, l'app mostra solo un modulo di login e non carica nessuna riga di codice, proteggendo il DB e le API.

2. **Deploy su Streamlit Community Cloud:**
   * Crea un deposito **privato** su GitHub e carica il tuo codice (escludendo i file delle password e delle chiavi API).
   * Collega l'account su Streamlit Community Cloud e importa il deposito privato.
   * Inserisci le chiavi segrete (API Groq, credenziali Supabase e Google JSON) nell'area sicura **"Advanced Settings -> Secrets"** del pannello di controllo di Streamlit.
   * Imposta l'applicazione come **"Private App"** e inserisci solo la tua email nella whitelist.

3. **Verifica Finale:**
   * Accedi al link della tua app dal browser dello smartphone, effettua il login e fai un test completo sul campo.
